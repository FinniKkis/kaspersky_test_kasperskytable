# kaspersky.table — Тестовое задание Kaspersky Lab

React-приложение «Корпоративный справочник»: управление сотрудниками и отделами.

## Запуск проекта

### Требования
- Node.js >= 18 / npm >= 9

### Установка
```bash
npm install
```

### Запуск (два терминала)

**Терминал 1 — JSON-сервер (API):**
```bash
npm run server   # http://localhost:3001
```

**Терминал 2 — React-приложение:**
```bash
npm run dev      # http://localhost:5173
```

## Структура проекта

```
src/
├── components/Layout.jsx       # Шапка + навигация + подвал
├── hooks/useApi.js             # Хуки useUsers, useGroups (json-server)
├── pages/
│   ├── WelcomePage.jsx         # страница приветствия
│   ├── UsersPage.jsx           # таблица сотрудников (без LLM)
│   └── GroupsPage.jsx          # карточки групп (с LLM)
├── App.jsx                     # BrowserRouter + Routes
└── index.css                   # CSS-переменные и reset
db.json                         # База данных json-server
```

## Бизнес-логика

- **Сотрудники** `/users` — таблица с поиском (имя/email/должность/отдел), сортировкой по любому столбцу (ASC/DESC), фильтром по статусу и отделу, добавлением через модальную форму с валидацией и удалением с подтверждением. Поиск и сортировка через `useMemo` — пересчёт только при изменении зависимостей.
- **Группы** `/groups` — карточки отделов с участниками, прогресс-бар активности, блок «без отдела».
- **API** — `json-server` на порту 3001, REST-эндпоинты `/users` и `/groups`.

## Выводы по проектированию UI

### Ручное проектирование
Полный контроль над каждым UX-решением: sortable-заголовки, inline-поиск, confirm-диалог, CSS Module. Требует чёткого понимания задачи и пользовательских сценариев, но результат предсказуем и консистентен в рамках дизайн-системы.

### Проектирование с LLM
LLM быстро генерирует визуально насыщенный макет: карточки, прогресс-бар, цветовые акценты на каждый отдел. Такой результат вручную занял бы в 3–4 раза больше времени. Однако код нужно адаптировать под существующую систему переменных и проверять логику. Оптимальная стратегия — LLM для быстрого прототипа каркаса, ручная доработка логики и стиля.

### Примечание
Все иконки и другие объекты сделаны с помощью JS+SVG с отрисовкой через код.
